# 测试用例生成技能

**用途：** 当用户提供产品需求文档(PRD)、技术规格文档，或描述功能需求时，自动激活此技能来生成全面的测试用例和测试场景矩阵，确保覆盖功能测试、边界测试和异常测试。

## 触发条件

激活此技能当用户：
- 上传或粘贴PRD/需求文档内容
- 请求"生成测试用例"、"分析测试场景"、"创建测试计划"
- 提供功能描述并要求测试覆盖
- 提到"边界测试"、"异常测试"、"验收标准"
- 手动使用 `/test-case-generation` 命令

## 工具依赖

- `read` - 读取本地文档文件（如用户提供文件路径）
- `write` - 输出测试计划文件（可选）
- 无工具时：基于用户提供的文本内容直接分析

## 工作流程

### 1. 需求解析
- 提取功能点、用户故事、验收标准
- 识别输入参数、业务规则、预期结果
- 标记边界条件和异常情况

### 2. 测试用例设计
- 为每个功能点设计：正常路径 + 边界值 + 异常情况（至少 3 个测试用例）
- 定义前置条件、测试步骤、验证点
- 标识测试依赖和数据准备

### 3. 输出生成
- 以表格形式呈现测试用例矩阵
- 提供Gherkin格式（可选，用于BDD）
- 标注优先级和执行顺序

## 输出模板

### 测试矩阵（默认格式）

```markdown
## 测试场景：[功能名称]

| 场景 ID | 类型 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|------|----------|----------|----------|--------|
| TC-001  | 功能 | ...      | ...      | ...      | P0     |
```

### Gherkin 格式（按需）

```gherkin
功能：[功能名称]
  作为 [用户角色]
  我希望 [功能目的]
  以便 [业务价值]

  场景：[场景描述]
    给定 [前置条件]
    当 [执行操作]
    那么 [预期结果]
```

### 测试覆盖报告

```markdown
## 覆盖度分析

- 功能点总数：X
- 已覆盖测试：Y
- 覆盖率：Z%
- 缺失覆盖：[列出未覆盖的功能点]
```

## 质量标准

- ✅ 完整性 - 每个功能点至少 3 个测试场景（正常 + 边界 + 异常）
- ✅ 准确性 - 测试准确反映需求，无歧义
- ✅ 可执行性 - 步骤清晰，结果可验证
- ✅ 可追溯性 - 测试场景可映射回原始需求

## 不适用场景

以下情况不激活此技能：
- 简单的理论咨询（如"什么是边界测试"）
- 非测试相关的需求讨论
- 用户明确表示只需要建议而非完整测试矩阵

## 最佳实践

1. 优先 P0 场景 - 先输出核心功能的正常路径测试
2. 明确判定标准 - 每个测试的预期结果必须清晰可验证
3. 数据独立 - 测试间尽量无依赖，可独立执行
4. 标注假设 - 如需求不明确，标注假设条件并请用户确认
